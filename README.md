# 家教平台简化版

这是一个本地静态版家教订单系统，数据保存在浏览器 `localStorage` 中。

## 页面入口

- `index.html`：入口页，提供老师端和中介端两个按钮。
- `teacher.html`：老师订单大厅，只显示 `active / 招募中` 订单，不展示家长电话、家长微信、后台备注、原始文本，也没有中介后台入口。
- `agent.html`：中介订单后台，用于录单、搜索、状态处理、操作日志、中介账号维护和数据备份。

## 老师端

- 支持按年级、科目、区域、关键词筛选，年级/科目/区域都可以多选叠加。
- 年级包含小学、初中、高中、新初一到新高三、成人、其他等。
- 科目包含语文、数学、英语、物理、化学、生物、历史、地理、政治、国际课程等，不使用“语数英”这类合并科目。
- 区域包含西安常用区域、线上和其他。
- 每个订单可复制地址，也可同步复制订单号和中介微信。
- 订单按每页 20 条显示，底部可通过数字页码、上一页和下一页跨页。

## 中介端

- 订单号不填时自动生成，填写时使用中介填写的编号。
- 重复订单号会要求修改。
- 发布前会提醒可能重复的订单，包括订单号、地址 + 年级 + 科目、家长电话/微信、原始文本相似。
- 暂时下架、恢复大厅、家长取消、删除订单等操作必须填写原因。
- 删除、家长取消、试课成功、清空演示数据需要二次确认。
- 后台保留原始文本、家长隐私字段、中介内部备注和操作日志。
- 中介后台顶部按订单录入、订单管理、账号安全三个页签展示，数据备份放在账号安全中。
- 订单文本使用标签、关键词和正则规则自动识别，不调用 AI；识别结果必须由中介核对修改后发布。
- 普通管理列表只显示招募中和下架中订单，每页 20 条；完成、取消和删除订单从工作台隐藏。
- 暂时下架必须填写接单老师联系方式，下架订单仍可通过订单号搜索并恢复上架或标记试课成功。

## 数据备份

中介端支持导出数据、导入数据和恢复演示数据。正式录入前建议先导出备份。

## 本地预览

可以直接双击打开 `index.html`，也可以启动静态服务器后访问：

```bash
python -m http.server 8765
```

然后打开：

- `http://127.0.0.1:8765/teacher.html`
- `http://127.0.0.1:8765/agent.html`
