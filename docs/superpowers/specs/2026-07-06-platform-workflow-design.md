# 家教平台分页、状态流程与云端化设计

## 目标

将当前静态演示版整理为可平滑迁移到云端数据库的产品原型。老师端保持公开只读，中介端需要登录；约 4000 条订单时仍能快速筛选、搜索和翻页。

## 页面与权限

- 老师端和中介端继续使用两个独立网址，老师端不存在任何中介后台入口。
- 老师端只读取 `active` 订单，支持年级、科目、区域多选和关键词搜索，每页 20 条，底部显示上一页、页码和下一页。
- 中介端顶部使用三个页签：`订单录入`、`订单管理`、`账号安全`，一次只显示一个工作区。
- 普通中介共享全部进行中订单，可以录入、搜索和操作；操作日志记录实际账号。
- 管理员可以创建、停用中介账号，并查看已完成、已取消、已删除订单及数据备份入口。

## 订单录入与规则识别

- 不使用 AI；采用字段标签、关键词词典和正则规则组合识别。
- 识别订单号、性别、年级、科目、区域、地址、成绩、上课时间、价格、老师要求、家长电话和家长微信。
- 一段文本可识别多个科目，但年级只能形成一个订单字段；无法确定时留空，不猜测。
- 识别结果进入可编辑表单，不自动发布；必填字段缺失或格式异常时标红并给出具体提示。
- 中介修改后再发布，数据库同时保存原始文本、最终字段和操作日志。
- 用真实订单样本建立解析测试集，每增加一种文案格式就补充回归测试。

## 状态流程

- `active`：老师大厅可见，中介订单管理默认可见。
- `paused`：大学生交定金或进入沟通/试课后下架；老师大厅立即不可见，中介仍可通过订单号搜索。下架时必须填写接单老师联系方式和原因。
- `paused → active`：家长不满意、试课失败或未接成时恢复上架，并记录原因。
- `paused → completed`：试课成功后结束；普通中介工作台立即消失，仅管理员历史记录可查。
- `active/paused → cancelled`：家长取消；普通中介工作台消失，仅管理员可查。
- `active/paused → deleted`：重复或录入错误；普通中介工作台消失，仅管理员可查。
- 完成、取消、删除必须二次确认；所有状态变化写入不可覆盖的操作日志。

## 订单管理与分页

- 默认只列出 `active` 和 `paused`，每页 20 条，底部使用数字页码、上一页和下一页。
- 搜索支持订单号、地址、年级、科目、家长联系方式和原始文本；优先支持完整订单号精确命中。
- 搜索和状态筛选变化后自动回到第 1 页；删除当前页最后一条后自动回退到有效页。
- 中介卡片删除复制地址、复制中介微信和联系文案按钮，只保留业务字段、编辑入口、状态操作与日志。
- 订单可在未结束状态下编辑；订单号修改需重新检查唯一性，关键字段修改写入日志。

## 云端数据边界

- 正式版采用云端 PostgreSQL、身份认证和服务端权限校验；浏览器 `localStorage` 仅保留筛选偏好，不保存正式订单。
- 老师端查询接口在服务端固定限定 `status = active`，不能依赖前端隐藏。
- 分页、搜索和排序由数据库完成，默认按发布时间倒序，并为状态、订单号、发布时间和常用搜索字段建立索引。
- 第一阶段先在现有原型中完成页面、分页、纯规则识别和状态行为；第二阶段再接入账号、数据库和部署。

## 验收标准

- 生成 4000 条测试订单后，老师端和中介端均只渲染当前 20 条，筛选和翻页无明显卡顿。
- `active → paused` 后老师端刷新即不可见，订单号搜索仍能在中介端找到。
- `paused → active` 后重新出现在老师端；`completed/cancelled/deleted` 对普通中介和老师均不可见。
- 管理员能查询历史订单并看到完整状态日志；普通中介不能访问账号创建和历史数据。
- 解析测试覆盖参考图片中的标签式文本及常见无标签文本，缺失字段不会被错误填充。
